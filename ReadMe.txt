Когда я работал в дежурной смене в департаменте сопровождения процессинговой системы - одной из моих функциональных обязанностей являлась фиксация "Sign in" - "Sign out" в Системе Учёта рабочего времени. 
В 23:55 необходимо было открыть в браузере Систему Учёта и поставить отметку "Ушёл". Система учитывала первые 3 часа ночной смены. 
А в 00:00 необходимо было снова открыть в браузере Систему Учёта и поставить отметку "Пришёл". Система учитывала оставшиеся 9 часов ночной смены.
Основная проблема такого ручного контроля - человеческий фактор (можно было забыть вовремя отметиться в Системе Учёта и тогда вечернее время до 23:55 не будет зачтено в систему мотивации сотрудника).

В качестве своего первого  пет-проекта на Java я решил написать скрипт, автоматизирующий вышеуказанные ручные действия (вход в Систему Учёта, поиск нужного раздела, нажатие кнопки "Check in", ожидание в 5-10 минут, повтор действий). 
Данный скрипт срабатывает автоматически в 23:55 по планировщику задач. Программа реализована с помощью фреймворка Selenium для браузера Opera. По сути, скрипт осуществляет действия в браузере по принципу кликера, находя нужные веб-элементы по локаторам fullXPath.

Описание текущего (ручного) процесса AS IS:
	1) пользователь открывает Систему Учёта https://apps.***.ru;
	2) пользователь аутентифицируется в Системе Учёта посредством ввода логопаса и переходит на вкладку учёта рабочего времени;
	3) в 23:55 пользователь нажимает кнопку Check in, Система фиксирует метку "Ушёл";
	4) в 00:00 пользователь повторяет шаги 2 и 3, Система фиксирует метку "Пришёл" (конец процесса).

Описание (автоматизированного) процесса TO BE: 
	1) планировщик заданий (на Windows) запускает исполняемый файл ровно в 23:55 (настройка планировщика выполняется разово);
	2) скрипт открывает ссылку на Систему Учёта https://apps.***.ru;
	3) скрипт считывает логин и пароль, хранимые в текстовом файле локально на компьютере, тем самым осуществляя аутентификацию пользователя в Системе Учёта; скрипт осуществляет переход на вкладку учёта рабочего времени;
	4) в 23:55 скрипт имитирует нажатие кнопки Check in, Система фиксирует метку "Ушёл"; после чего происходит неявное ожидание в 300000 милисекунд (5 минут);
	5) в 00:00 скрипт имитирует нажатие кнопки Check in, Система фиксирует метку "Пришёл" (конец процесса).
